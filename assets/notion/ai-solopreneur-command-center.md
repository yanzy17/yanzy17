## SOP Template (one per automation)

### Automation #1 — Content Multiplier — SOP

**How it works:**
1. You drop a 1,000-word essay into Notion's Content Pipeline DB.
2. Status set to "Drafting."
3. Make.com fires 6 ChatGPT calls in sequence.
4. Each call writes to a different Notion property (LinkedIn, Twitter Thread, etc.).
5. Status auto-updates to "Multiplied."
6. You review (5 min), approve, schedule via Buffer.

**Setup time:** 90 minutes (one-time)

**Maintenance:**
- Weekly: drop 2 essays in
- Monthly: review/update prompts (10 min)
- Quarterly: refresh voice samples in Brand DNA

**Troubleshoot:**
- AI tone drifting → update Voice File examples
- Missed scheduling → check Buffer API connection
- Output too generic → tighten prompt instructions

## 🎣 HOOK GENERATORS

### #1 — Multi-Hook Generator
```
Generate 10 hooks for a [PLATFORM] post about [TOPIC].

Use these formulas (mix freely):
- Contrarian: "Everyone says X. They're wrong."
- Number-led: "3 mistakes [audience] makes with X"
- Curiosity gap: "Why most [X] fail isn't what you think"
- Specific story: "I lost $X doing Y. Here's what I learned."
- Prediction: "In 12 months, X will be obsolete."

Each hook ≤ 12 words. Output as numbered list.
```

```
Write 7 Twitter opening lines for a thread about [TOPIC].

```

## 💬 CUSTOMER SUPPORT

### #10 — Polite Refund Reply
```
Customer is requesting a refund.
Their message: [PASTE MESSAGE]
My refund policy: 30-day, no questions asked.

Reply:
- Empathetic but not over-apologetic
- Confirm refund processing (3-5 business days)
- Ask 1 question to learn what didn't work (optional)
- No fake offers to "make it right"
- Sign off warmly

Max 70 words.
```

### #11 — Angry Customer De-escalation
```
Customer is angry: [PASTE MESSAGE]

Reply:
- Acknowledge their frustration in first sentence
- Take ownership (not blame the customer)
- State the specific fix (with timeline)
- Offer one tangible gesture (refund / extra service / discount)
- End with a direct line to me ("reply if not resolved by [date]")

Tone: calm, owner, not defensive.
```
